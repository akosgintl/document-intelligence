# Confidence Threshold is separate, mutable per-Type config, not part of the immutable Schema

Schema versions are explicitly immutable once any Document has been processed against them, but Confidence Threshold is an operational tuning parameter — how forgiving the pipeline is — not a statement about document structure. Baking it into the Schema JSON would couple the two, forcing a new Schema version just to retune review sensitivity. We chose to store Confidence Threshold as separate metadata attached to the Document Type, set by the platform operator and mutable independently of Schema versioning. It's required at registration with no system-wide default — consistent with this project's existing preference for explicit configuration over silent defaults (e.g. `unclassified` as an explicit outcome rather than a forced best-effort guess) — and one value is shared by both Classification and Extraction gating rather than split per-concern, since nothing yet demonstrates the two decisions need different sensitivity.
