# Confidence is computed via self-reported certainty, not logprobs or sampling

This system's Confidence score has to come from somewhere — investigated in `docs/research/confidence-via-anthropic-api.md` — and we adopted self-reported certainty (the model emits its own confidence value as a field in the same Classification/Extraction tool-use call) as the sole mechanism for v1. Token-level logprobs are ruled out entirely, not just deprioritized: Anthropic's Messages API doesn't expose them at all. Best-of-N sampling agreement is a real, Anthropic-documented technique, but we deferred it — adopting it now would mean guessing at an escalation margin, a sample count N, and disagreement-handling logic (which value wins when samples disagree) with zero calibration data to ground any of those choices. Self-reported confidence's own calibration is unconfirmed by any Anthropic source either; empirical validation against a labeled holdout set is necessary before the Confidence Threshold gate can be fully trusted, tracked as follow-up work rather than blocking this decision.
